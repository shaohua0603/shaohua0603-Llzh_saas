# 菜单配置功能开发计划

## 一、项目概述

### 1.1 项目背景
根据蓝领智汇SaaS系统的PC端功能规格说明和UI设计规范，需要开发菜单配置功能，该功能位于系统管理模块下，用于管理系统的一级、二级、三级菜单以及页面按钮权限。

### 1.2 项目目标
- 实现菜单的增删改查功能
- 实现菜单的树形展示和层级管理
- 实现菜单权限控制（数据权限、菜单权限、按钮权限）
- 严格遵循项目开发规范和UI设计规范
- 确保代码质量和可维护性

### 1.3 技术栈
- **前端**: Vue 3 + TypeScript + Element Plus + Vue Router + Pinia
- **后端**: Spring Boot + MyBatis Plus + MySQL + Redis
- **工具**: Vite + ESLint + Prettier

---

## 二、需求分析总结

### 2.1 核心功能需求
根据需求分析报告，菜单配置功能包括以下核心需求：

#### 2.1.1 基础功能
- 菜单列表展示（树形结构）
- 菜单新增（支持一级、二级、三级菜单）
- 菜单编辑（修改菜单信息）
- 菜单删除（需二次确认，检查子菜单）
- 菜单查询（按菜单名称、菜单类型筛选）

#### 2.1.2 菜单字段定义
| 字段名称 | 字段类型 | 必填 | 说明 |
|----------|----------|------|------|
| 菜单名称 | 文本 | 是 | 菜单显示名称 |
| 菜单路径 | 文本 | 是 | 菜单路由路径 |
| 菜单图标 | 图标选择 | 否 | 菜单显示图标 |
| 菜单类型 | 枚举 | 是 | 目录/菜单/按钮 |
| 业务代码 | 文本 | 是 | 业务标识，用于权限关联 |
| 上级菜单 | 树形选择 | 否 | 父级菜单 |
| 排序序号 | 数字 | 是 | 菜单显示顺序 |
| 菜单状态 | 枚举 | 是 | 启用/禁用 |
| 菜单描述 | 多行文本 | 否 | 菜单功能说明 |

#### 2.1.3 业务规则
- 菜单层级不超过4级
- 一级菜单固定为9个核心业务领域
- 菜单与业务代码关联，用于权限控制
- 删除菜单前需检查是否存在子菜单
- 业务代码必须唯一

### 2.2 UI设计要求
- 列表页面使用树形表格展示菜单
- 表单页面使用卡片式布局
- 功能按钮栏统一放置在查询条件和列表之间
- 严格遵循UI设计规范的布局、样式、交互要求

### 2.3 权限控制要求
- 数据权限：全部数据/本部门数据/本部门及以下/本人数据/自定义
- 菜单权限：根据岗位配置动态生成路由菜单
- 按钮权限：根据岗位配置控制按钮显示/隐藏

## 三、技术方案

### 3.1 前端技术方案

#### 3.1.1 目录结构
```
src/
├── views/
│   └── admin/
│       └── system/
│           └── menu/
│               ├── MenuConfig.vue          # 菜单配置列表页
│               ├── MenuForm.vue            # 菜单表单页
│               └── components/
│                   ├── MenuTreeTable.vue   # 菜单树形表格
│                   ├── MenuIconPicker.vue  # 菜单图标选择器
│                   └── MenuPathInput.vue   # 菜单路径输入
├── api/
│   └── system/
│       └── menuApi.ts                      # 菜单API接口
├── types/
│   └── system/
│       └── menuTypes.ts                    # 菜单类型定义
├── stores/
│   └── system/
│       └── menuStore.ts                    # 菜单状态管理
└── utils/
    ├── permission.ts                       # 权限工具
    └── validate.ts                         # 验证工具
```

#### 3.1.2 核心技术点
- 使用Element Plus的Table组件实现树形表格
- 使用Element Plus的Tree组件实现上级菜单选择
- 使用Element Plus Icons实现图标选择器
- 使用Vue Router实现页面跳转
- 使用Pinia实现状态管理
- 使用自定义指令实现按钮权限控制

### 3.2 后端技术方案

#### 3.2.1 数据库设计
```sql
CREATE TABLE sys_menu (
  id BIGINT PRIMARY KEY AUTO_INCREMENT COMMENT '菜单ID',
  parent_id BIGINT DEFAULT 0 COMMENT '父菜单ID',
  menu_name VARCHAR(50) NOT NULL COMMENT '菜单名称',
  menu_type VARCHAR(10) NOT NULL COMMENT '菜单类型:目录/菜单/按钮',
  menu_icon VARCHAR(50) COMMENT '菜单图标',
  business_code VARCHAR(50) NOT NULL UNIQUE COMMENT '业务代码',
  menu_path VARCHAR(200) COMMENT '菜单路径',
  sort_order INT DEFAULT 0 COMMENT '排序序号',
  menu_status VARCHAR(10) DEFAULT 'ENABLED' COMMENT '菜单状态',
  menu_desc VARCHAR(200) COMMENT '菜单描述',
  tenant_id BIGINT NOT NULL COMMENT '租户ID',
  create_time DATETIME DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  update_time DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  create_by VARCHAR(50) COMMENT '创建人',
  update_by VARCHAR(50) COMMENT '更新人',
  INDEX idx_parent_id (parent_id),
  INDEX idx_business_code (business_code),
  INDEX idx_tenant_id (tenant_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='系统菜单表';
```

#### 3.2.2 API接口设计
- GET /api/v1/system/menus - 获取菜单列表
- GET /api/v1/system/menus/tree - 获取菜单树
- GET /api/v1/system/menus/{id} - 获取菜单详情
- POST /api/v1/system/menus - 创建菜单
- PUT /api/v1/system/menus/{id} - 更新菜单
- DELETE /api/v1/system/menus/{id} - 删除菜单
- GET /api/v1/system/menus/check-code - 验证业务代码唯一性
