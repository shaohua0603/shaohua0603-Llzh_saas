### 1.2 功能需求

根据 `pc-feature-spec.md` 第154-155行，字典管理的功能需求：

```
#### 字典管理
-**功能**：按照数据权限展示字典管理，包含：字典名称、字典类型、字典编码、字典值、字典说明，支持增删改查
```

### 1.3 设计规范要求

根据 `ui-design-spec.md` 和 `project_rules.md`，字典管理需要遵循：

1. **列表页面规范**（第24节）：
   - 统一使用 CommonTable 组件
   - 支持列宽调整、字段显示/隐藏、自定义列表
   - 支持悬浮提示、文本截断、多选和全选
   - 支持加载状态和空数据提示
   - 支持分页、排序、筛选

2. **数据权限配置规范**（第22节）：
   - 支持全部数据、本部门数据、本部门及以下、本人数据、自定义数据权限
   - 所有列表查询自动应用数据权限过滤

3. **操作按钮规范**（第27节）：
   - 新增/编辑：跳转到新页面
   - 删除：弹窗二次确认
   - 导入/导出：弹窗导入表单/直接下载文件

4. **页面布局规范**（第28节）：
   - 功能按钮栏统一放置在查询条件区域和列表区域之间
   - 布局顺序：查询条件区域 → 功能按钮栏 → 列表区域 → 分页区域

## 二、需求分析

### 2.1 功能需求拆解

#### 2.1.1 列表页面功能

**基础功能：**
1. 字典列表展示
   - 字典名称
   - 字典类型
   - 字典编码
   - 字典值（显示字典值的数量或预览）
   - 字典说明
   - 创建时间
   - 操作（查看详情、编辑、管理值、删除）

2. 搜索功能
   - 按字典名称搜索
   - 按字典编码搜索
   - 按字典类型筛选

3. 排序功能
   - 支持按创建时间排序
   - 支持按字典名称排序

4. 分页功能
   - 支持自定义每页显示条数
   - 显示总记录数

5. 表格功能（使用CommonTable）
   - 列宽调整
   - 字段显示/隐藏
   - 自定义列表
   - 悬浮提示
   - 文本截断
   - 多选和全选

6. 功能按钮栏
   - 新增字典
   - 导入字典
   - 导出字典
   - 批量删除

#### 2.1.2 字典表单功能（弹窗方式）

**字典表单：**
1. 基本信息
   - 字典名称（必填）
   - 字典类型（必填，下拉选择）
   - 字典编码（必填，自动生成或手动输入）
   - 字典说明（选填）

2. 字典值管理
   - 字典值列表
   - 字典值增删改
   - 字典值排序
   - 字典值状态

3. 表单验证
   - 字典名称必填
   - 字典类型必填
   - 字典编码必填且唯一
   - 字典值至少需要一个

4. 操作按钮
   - 保存
   - 取消

#### 2.1.3 数据权限功能

**权限控制：**
1. 根据登录账户的数据权限展示字典
   - 全部数据：查看所有字典
   - 本部门数据：仅查看本部门创建的字典
   - 本部门及以下：查看本部门及下属部门创建的字典
   - 本人数据：仅查看本人创建的字典
   - 自定义：查看自定义部门范围内的字典

2. 操作权限控制
   - 新增：根据岗位权限
   - 编辑：根据岗位权限和数据权限
   - 删除：根据岗位权限和数据权限

### 2.2 技术需求

#### 2.2.1 前端技术需求

1. **组件开发**
   - 使用 Vue 3 Composition API
   - 使用 `<script setup>` 语法
   - 使用 Element Plus 组件库
   - 使用 CommonTable 组件（替换现有 el-table）
   - 使用 TypeScript 类型定义

2. **状态管理**
   - 使用现有的 dictionary store 或组件内部状态

3. **路由管理**
   - 现有路由已配置完善

4. **API服务**
   - 现有的 dictionaryService.ts 已实现完整

5. **类型定义**
   - 现有的 dictionary.ts 已完善

6. **Mock数据**
   - 需要确认 mock 接口是否完整
