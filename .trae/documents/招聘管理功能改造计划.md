# 招聘管理功能改造计划

## 一、需求概述

根据《PC端功能说明文档》和《UI设计规范》，招聘管理模块需要实现以下功能：

### 1.1 招聘需求功能
- 展示所有当前登录账号发布的招聘需求
- 展示工厂指定对应劳务公司的代招的需求
- 展示平台的全部招聘需求
- 支持跳转到对应招聘需求的详情页面

### 1.2 简历管理功能
- 展示所有当前登录账号发布需求收到的简历
- 支持跳转到对应简历的详情页面

## 四、技术规范

### 4.1 路由路径规范（按照项目规范）
- 招聘需求列表：`/labor-company/recruitment`
- 招聘需求详情：`/labor-company/recruitment/detail/:id`
- 招聘需求新增：`/labor-company/recruitment/add`
- 招聘需求编辑：`/labor-company/recruitment/edit/:id`
- 简历管理列表：`/labor-company/recruitment/resume`
- 简历详情：`/labor-company/recruitment/resume/:id`

### 4.2 组件使用规范
- 使用 CommonTable 组件进行列表展示
- 使用 Element Plus 表单组件
- 使用 Element Plus 弹窗组件进行确认操作

### 4.3 数据权限规范
- 列表查询需要根据登录账户的数据权限过滤
- Tab切换时保持数据权限过滤
