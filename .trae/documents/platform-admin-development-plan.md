### 1.3 核心技术要求

#### 前端技术要求
- **框架**：Vue 3 + TypeScript + Composition API
- **UI组件库**：Element Plus
- **路由**：Vue Router（懒加载、路由守卫）
- **状态管理**：Pinia
- **表格组件**：封装统一表格组件（排序、搜索、列宽调整、悬浮提示、文本截断、分页、多选、自定义列表、字段展示设置）
- **日期时间组件**：Element Plus日期时间选择器（中文友好）
- **布局**：统一PC端布局（顶部一级菜单、左侧二级/三级菜单、四级页面标签栏）
- **响应式**：适配移动端、平板、桌面端

#### 后端技术要求
- **API设计**：RESTful风格
- **响应格式**：统一响应格式（code、message、data、timestamp）
- **数据权限**：支持全部数据、本部门数据、本部门及以下、本人数据、自定义
- **审批流程**：支持审批流程配置、审核记录、状态流转
- **分页**：统一分页参数（page、pageSize、sortField、sortOrder）
- **文件上传**：支持图片、文件上传

## 四、开发规范要求

### 4.1 前端开发规范

#### 4.1.1 代码风格
- 使用 2 个空格缩进
- 每行代码长度不超过 120 个字符
- 使用单引号或反引号
- 使用小驼峰命名法（变量、函数）
- 使用大驼峰命名法（组件）
- 使用中文注释

#### 4.1.2 组件开发
- 使用 Vue 3 Composition API
- 使用 `<script setup>` 语法
- 组件结构清晰，逻辑分离
- 遵循单一职责原则
- 组件可复用性设计

#### 4.1.3 样式管理
- 使用 scoped 样式
- 避免使用 !important
- 合理使用 CSS 变量
- 响应式设计适配不同屏幕

#### 4.1.4 路由管理
- 使用懒加载减少初始加载时间
- 添加路由元信息
- 配置路由守卫
- 统一路由跳转方式

#### 4.1.5 状态管理
- 小型状态：使用组件内部状态
- 中型状态：使用 provide/inject
- 大型状态：使用 Pinia

#### 4.1.6 数据处理
- 使用异步函数处理数据获取
- 使用 try-catch 捕获错误
- 实现加载状态管理
- 数据缓存策略

#### 4.1.7 TypeScript 使用
- 为所有变量添加类型
- 为函数参数和返回值添加类型
- 使用接口定义复杂类型
- 使用泛型提高代码复用性

### 4.2 后端开发规范

#### 4.2.1 API 设计
- 遵循 RESTful 设计原则
- 使用 HTTP 方法表示操作类型
- URL 使用小写字母和连字符
- 版本号放在 URL 中
- 统一响应格式
- 统一错误处理
- 支持分页、排序、筛选
- 支持批量操作
- 支持数据权限过滤

#### 4.2.2 响应格式
```typescript
interface ApiResponse<T> {
  code: number;
  message: string;
  data: T;
  timestamp: number;
}

interface PageResponse<T> {
  code: number;
  message: string;
  data: {
    list: T[];
    total: number;
    page: number;
    pageSize: number;
  };
  timestamp: number;
}
```

#### 4.2.3 响应码规范
| 响应码 | 编码 | 说明 |
|--------|------|------|
| 200 | SUCCESS | 成功 |
| 201 | CREATED | 创建成功 |
| 400 | BAD_REQUEST | 请求参数错误 |
| 401 | UNAUTHORIZED | 未授权 |
| 403 | FORBIDDEN | 无权限 |
| 404 | NOT_FOUND | 资源不存在 |
| 500 | INTERNAL_ERROR | 服务器错误 |

#### 4.2.4 数据库设计
- 使用统一的命名规范
- 为所有表添加创建时间和更新时间字段
- 为所有表添加软删除字段
- 合理设计索引
- 合理设计外键关系

#### 4.2.5 数据权限
- 支持全部数据、本部门数据、本部门及以下、本人数据、自定义
- 所有列表查询自动应用数据权限过滤
- 数据权限支持组合配置

### 4.3 UI 设计规范

#### 4.3.1 页面布局
- 顶部导航栏高度：60px
- 左侧菜单宽度：200px（展开）、60px（折叠）
- 四级导航栏高度：40px
- 内容区域内边距：20px
- 卡片间距：20px

#### 4.3.2 列表页面
- 查询条件区域 → 功能按钮栏 → 列表区域 → 分页区域
- 功能按钮栏使用 `action-bar` 样式
- 集成统一表格组件
- 支持列宽调整、字段显示/隐藏、自定义列表
- 支持悬浮提示、文本截断
- 支持多选和全选
- 支持加载状态和空数据提示

#### 4.3.3 表单页面
- 必填项标识清晰（红色星号）
- 字段分组合理
- 验证规则完善
- 提示文字友好
- 支持表单重置
- 支持表单保存草稿
- 支持表单联动

#### 4.3.4 详情页面
- 信息分组展示，层次清晰
- 重要信息优先展示
- 支持折叠/展开
- 操作记录完整展示
- 审核记录时间线展示
- 附件支持预览和下载

#### 4.3.5 操作按钮
- 新增/编辑：跳转到新页面
- 删除：弹窗二次确认
- 审核：弹窗审核表单
