# 页签标题统一规范

## 1. 概述

本文档旨在规范蓝领智汇平台中4级导航页签标题的命名和管理，确保页签标题与对应页面名称保持一致，提升用户体验和系统一致性。

## 2. 页签标题来源

页签标题应与以下来源保持一致：

- **菜单名称**：页签标题应与左侧菜单或顶部菜单中的对应菜单项名称完全一致
- **页面标题**：页签标题应与页面内部的标题保持一致
- **路由配置**：页签标题应在布局文件的路径映射中明确定义

## 3. 路径映射规则

### 3.1 基本规则

- **唯一映射**：每个路由路径应对应唯一的页签标题
- **路径一致性**：路径映射应与实际路由路径完全匹配
- **层次清晰**：路径映射应反映页面的层次结构
- **避免重复**：严禁在同一个布局文件中出现重复的路径映射

### 3.2 路径格式

- **列表页面**：`/module/list` → 模块名称
- **详情页面**：`/module/detail/:id` → 模块详情
- **编辑页面**：`/module/edit/:id` → 编辑模块
- **新增页面**：`/module/add` → 新增模块

## 5. 命名规范

### 5.1 通用规则

- **一致性**：页签标题应与菜单项名称完全一致
- **简洁性**：标题应简洁明了，避免过长
- **准确性**：标题应准确反映页面内容和功能
- **标准化**：使用统一的术语和命名风格

### 5.2 具体规范

- **列表页面**：使用“管理”、“列表”等后缀，如“简历管理”、“工人列表”
- **详情页面**：使用“详情”后缀，如“简历详情”、“招聘需求详情”
- **编辑页面**：使用“编辑”前缀，如“编辑接送”、“编辑初步面试”
- **新增页面**：使用“新增”前缀，如“新增接送”、“新增面试邀约”

## 6. 维护与更新

### 6.1 新增页面

- 当新增页面时，应在对应布局文件的 `titleMap` 中添加路径映射
- 确保路径映射与菜单项名称保持一致
- 测试页签标题显示是否正确

### 6.2 修改页面

- 当修改页面名称时，应同时更新对应布局文件中的路径映射
- 确保所有相关的路径映射都得到更新

### 6.3 删除页面

- 当删除页面时，应从对应布局文件的 `titleMap` 中移除对应的路径映射
- 确保不会留下无用的映射项

## 7. 测试与验证

### 7.1 测试方法

1. 启动开发服务器：`npm run dev`
2. 登录系统，导航到各个模块
3. 检查页签标题是否与菜单名称一致
4. 检查详情、编辑、新增页面的页签标题是否正确

### 7.2 验证标准

- 页签标题与菜单项名称完全一致
- 页签标题与页面内部标题一致
- 路径映射无重复
- 所有页面都有对应的页签标题

## 8. 常见问题与解决方案

### 8.1 页签标题显示为“页面”

**原因**：路径映射中缺少对应的路径
**解决方案**：在对应布局文件的 `titleMap` 中添加路径映射

### 8.2 页签标题与菜单名称不一致

**原因**：路径映射中的标题与菜单名称不匹配
**解决方案**：更新路径映射中的标题，使其与菜单名称一致

### 8.3 重复路径映射

**原因**：在同一个布局文件中出现了相同的路径映射
**解决方案**：移除重复的路径映射，确保每个路径只有一个映射

## 9. 最佳实践

1. **统一管理**：所有页签标题应在布局文件的 `titleMap` 中统一管理
2. **定期检查**：定期检查路径映射的完整性和一致性
3. **文档同步**：当修改页签标题时，同步更新本规范文档
4. **团队协作**：团队成员应遵循统一的命名规范和路径映射规则

## 10. 结论

通过遵循本规范，可以确保蓝领智汇平台中4级导航页签标题与对应页面名称保持统一，提升用户体验和系统一致性。各开发人员应严格遵守本规范，确保页签标题的准确性和一致性。
