## 3. 核心功能特性

### 3.1 附件配置管理
- **灵活的附件配置**:支持5种附件类型(图片、文件、视频、音频、文档)
- **智能限制**:根据配置自动应用文件类型、大小、批量上传等限制
- **模板管理**:支持附件模板关联和下载
- **状态管理**:支持启用/禁用配置
- **批量操作**:支持批量删除

### 3.2 附件上传功能
- **智能上传**:根据配置自动选择上传方式
  - 小文件(≤10MB):普通上传
  - 大文件(>10MB):分片上传
- **分片上传**:支持大文件分片上传,提高上传成功率
- **断点续传**:支持断点续传,网络中断后可继续上传
- **并发控制**:支持并发上传,提高上传效率
- **自动重试**:上传失败自动重试,最多3次
- **进度显示**:实时显示上传进度
- **模板下载**:支持附件模板下载

### 3.3 附件预览功能
- **图片预览**:支持缩放、旋转、全屏查看
- **视频预览**:支持在线播放、播放控制
- **音频预览**:支持在线播放、播放控制
- **文档预览**:支持PDF、Word、Excel等格式预览
- **下载功能**:支持文件下载

### 3.4 附件模板管理
- **模板上传**:支持模板文件上传
- **模板编辑**:支持模板信息编辑
- **模板下载**:支持模板文件下载
- **下载统计**:显示模板下载次数
- **批量删除**:支持批量删除模板

## 4. 技术实现亮点

### 4.1 分片上传实现
- **文件分片**:将大文件分成多个小片(默认5MB一片)
- **并发上传**:支持并发上传多个分片(默认3个并发)
- **断点续传**:检查已上传分片,只上传未完成分片
- **自动重试**:上传失败自动重试,最多3次
- **指数退避**:重试延迟指数增长(1s, 2s, 4s)
- **进度计算**:实时计算上传进度

### 4.2 智能上传策略
- **文件大小判断**:根据文件大小自动选择上传方式
- **阈值配置**:可配置分片上传阈值(默认10MB)
- **降级处理**:分片上传失败时降级为普通上传

### 4.3 组件化设计
- **SmartAttachmentUpload**:智能上传组件,集成所有上传功能
- **AttachmentPreview**:预览组件,支持多种文件类型
- **可复用性**:组件设计考虑了可复用性
- **易用性**:提供简单的API接口

### 4.4 类型安全
- **TypeScript**:使用TypeScript提供类型安全
- **接口定义**:完整的接口定义和类型注解
- **类型检查**:编译时类型检查,减少运行时错误

### 6.1 访问附件配置管理页面

- **功能**:查看、上传、编辑、删除附件模板

### 6.3 在业务页面中使用SmartAttachmentUpload组件

```vue
<template>
  <SmartAttachmentUpload
    v-model="fileList"
    :config-id="configId"
    :business-id="businessId"
    :business-type="businessType"
    @success="handleSuccess"
    @error="handleError"
    @preview="handlePreview"
  />
</template>

<script setup lang="ts">
import SmartAttachmentUpload from '@/components/SmartAttachmentUpload.vue'
import { ref } from 'vue'

const fileList = ref([])
const configId = ref('1')
const businessId = ref('worker-001')
const businessType = ref('worker')

const handleSuccess = (response, file, fileList) => {
  console.log('上传成功', response, file, fileList)
}

const handleError = (error, file, fileList) => {
  console.error('上传失败', error, file, fileList)
}

const handlePreview = (file) => {
  console.log('预览文件', file)
}
</script>
```
