# 租户管理模块开发总结

## 开发时间
2026-02-26

## 模块概述
平台管理端第二阶段租户管理模块，用于管理平台上的租户（劳务公司和工厂）信息。

## 功能清单

### 1. 租户信息管理
- 租户列表展示（租户名称、统一社会信用代码、营业执照、管理员姓名、手机号、联系人姓名、联系人手机号、套餐名称、创建时间、状态）
- 租户搜索（关键词搜索：租户名称/信用代码/手机号）
- 租户筛选（按套餐、按状态）
- 租户新增（跳转新页面）
- 租户编辑（跳转新页面）
- 租户删除（弹窗二次确认）
- 租户启用/禁用
- 营业执照预览
- 批量删除
- 导出功能

## 技术实现

### 1. 类型定义
```typescript
// 租户状态枚举
enum TenantStatus {
  ENABLED = 'enabled',
  DISABLED = 'disabled'
}

// 租户类型枚举
enum TenantType {
  LABOR_COMPANY = 'labor_company',
  FACTORY = 'factory'
}

// 租户信息接口
interface Tenant {
  id: string
  tenantName: string
  creditCode: string
  businessLicense: string
  adminName: string
  adminPhone: string
  contactName: string
  contactPhone: string
  packageId: string
  packageName: string
  tenantType: TenantType
  status: TenantStatus
  createdAt: string
  updatedAt: string
  createdBy: string
  updatedBy: string
}
```

### 2. API接口
- `GET /system/tenants` - 获取租户列表
- `GET /system/tenants/:id` - 获取租户详情
- `POST /system/tenants` - 创建租户
- `PUT /system/tenants/:id` - 更新租户
- `DELETE /system/tenants/:id` - 删除租户
- `DELETE /system/tenants/batch` - 批量删除租户
- `GET /system/tenants/export` - 导出租户列表
- `PUT /system/tenants/:id/enable` - 启用租户
- `PUT /system/tenants/:id/disable` - 禁用租户
- `GET /system/tenants/check-name` - 检查租户名称是否重复
- `GET /system/tenants/check-credit-code` - 检查统一社会信用代码是否重复
- `GET /system/packages/options` - 获取套餐下拉选项

### 3. 状态管理
使用Pinia进行状态管理，包含以下状态和方法：
- `tenantList` - 租户列表
- `currentTenant` - 当前租户详情
- `total` - 总条数
- `loading` - 加载状态
- `packageOptions` - 套餐选项
- `fetchTenantList` - 获取租户列表
- `fetchTenantDetail` - 获取租户详情
- `createTenant` - 创建租户
- `updateTenant` - 更新租户
- `deleteTenant` - 删除租户
- `batchDeleteTenants` - 批量删除租户
- `exportTenants` - 导出租户列表
- `enableTenant` - 启用租户
- `disableTenant` - 禁用租户
- `checkTenantName` - 检查租户名称
- `checkCreditCode` - 检查统一社会信用代码
- `fetchPackageOptions` - 获取套餐选项

### 4. 页面功能

#### 租户信息列表页面（TenantInfo.vue）
- 查询条件：关键词、套餐名称、租户状态
- 功能按钮：新增、批量删除、导出
- 列表展示：租户名称、统一社会信用代码、营业执照、管理员姓名、管理员手机号、联系人姓名、联系人手机号、套餐名称、创建时间、状态
- 操作按钮：详情、编辑、启用/禁用、删除
- 分页功能
- 营业执照预览对话框

#### 租户表单页面（TenantForm.vue）
- 基本信息：租户名称、统一社会信用代码、租户类型、套餐、营业执照
- 管理员信息：管理员姓名、管理员手机号
- 联系人信息：联系人姓名、联系人手机号
- 状态设置：状态
- 表单验证：必填项、手机号格式、统一社会信用代码格式
- 支持新增和编辑两种模式

#### 租户详情页面（TenantDetail.vue）
- 基本信息：租户名称、统一社会信用代码、租户类型、套餐名称、状态、创建时间、更新时间、创建人
- 营业执照：图片预览
- 管理员信息：管理员姓名、管理员手机号
- 联系人信息：联系人姓名、联系人手机号
- 操作按钮：编辑、返回

## 开发规范遵循

### 代码风格
- 使用2个空格缩进
- 每行代码长度不超过120个字符
- 使用单引号或反引号
- 使用小驼峰命名法（变量、函数）
- 使用大驼峰命名法（组件）
- 使用中文注释

### 组件开发
- 使用Vue 3 Composition API
- 使用`<script setup>`语法
- 组件结构清晰，逻辑分离
- 遵循单一职责原则

### 样式管理
- 使用scoped样式
- 避免使用!important
- 合理使用CSS变量
- 响应式设计适配不同屏幕

### 路由管理
- 使用懒加载减少初始加载时间
- 添加路由元信息
- 配置路由守卫
- 统一路由跳转方式

### 状态管理
- 使用Pinia进行状态管理
- 状态结构清晰
- 避免状态冗余
- 状态更新可预测

### 数据处理
- 使用异步函数处理数据获取
- 使用try-catch捕获错误
- 实现加载状态管理
- 数据缓存策略

### TypeScript使用
- 为所有变量添加类型
- 为函数参数和返回值添加类型
- 使用接口定义复杂类型
- 使用泛型提高代码复用性

### API设计
- 遵循RESTful设计原则
- 使用HTTP方法表示操作类型
- URL使用小写字母和连字符
- 版本号放在URL中
- 统一响应格式
- 统一错误处理
- 支持分页、排序、筛选
- 支持批量操作
- 支持数据权限过滤

### 响应码规范
- 200（SUCCESS）- 成功
- 201（CREATED）- 创建成功
- 400（BAD_REQUEST）- 请求参数错误
- 401（UNAUTHORIZED）- 未授权
- 403（FORBIDDEN）- 无权限
- 404（NOT_FOUND）- 资源不存在
- 500（INTERNAL_ERROR）- 服务器错误

- 顶部导航栏高度60px

- 四级导航栏高度40px
- 内容区域内边距20px
- 卡片间距20px

- 支持列宽调整

- 支持自定义列表

- 支持文本截断
- 支持多选和全选
- 支持加载状态和空数据提示

### 表单页面
- 必填项标识清晰（红色星号）
- 字段分组合理
- 验证规则完善
