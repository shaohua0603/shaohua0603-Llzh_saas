# 蓝领智汇平台页面规范合规性检查计划

## 检查目标
根据项目规范文档，逐一检查当前项目的每一个页面是否与规范相符，如不相符需要进行逐一罗列。

## 检查范围
- 所有页面组件
- 布局结构
- 表单设计
- 表格组件
- 操作按钮
- 响应式设计
- 样式规范

## 检查团队
- 项目经理
- 产品经理
- 前端开发人员

## 检查流程
1. 梳理所有页面清单
2. 对照项目规范文档制定检查项
3. 逐一检查每个页面
4. 记录不符合规范的问题
5. 生成检查报告

## 详细检查计划

### 1. 页面清单梳理
- **管理员端**：Dashboard、Home、Recruitment、Tenants、Users、Packages、IdleWorkers 等
- **劳务公司端**：Dashboard、Home、Recruitment、Workers、Employees、Departments 等
- **工厂端**：Dashboard、Home、Recruitment、Workers、Departments 等
- **工人端**：Home、Profile、Attendance、Salary、Leave 等
- **公共页面**：Login、Register、ForgotPassword 等

### 2. 规范检查项

#### 2.1 布局设计规范
- 布局结构是否符合规范（底部导航栏/左侧菜单+顶部导航）
- 导航设计是否符合规范
- 响应式布局是否适配不同屏幕
- PC端统一菜单结构是否符合规范
- 侧边栏折叠/展开功能是否实现
- 菜单图标规范是否符合要求
- 菜单交互规范是否符合要求

#### 2.2 页面布局规范
- 页面标题展示是否符合规范
- 统一页面布局结构是否符合规范（查询条件区域 → 功能按钮区域 → 数据统计区域 → 列表展示区域）
- 布局顺序是否正确
- 功能按钮栏位置是否符合规范
- 功能按钮栏样式是否符合规范
- 数据统计区域样式是否符合规范
- 响应式适配是否符合规范
- 布局组件padding值是否符合规范

#### 2.3 列表页面设计规范
- 列表页面配置是否符合规范
- 列配置是否符合规范
- 列表页面功能要求是否实现（排序、搜索、列宽调整、悬浮提示、文本截断、分页、多选等）
- 列表页面操作按钮位置是否符合规范

#### 2.4 详情页面设计规范
- 详情页面配置是否符合规范
- 信息分组是否符合规范
- 字段配置是否符合规范
- 详情页面设计原则是否遵循
- 底部悬浮按钮规范是否符合要求
- 详情页面跳转方式是否符合规范

#### 2.5 表单设计规范
- 表单配置是否符合规范
- 表单字段类型是否符合规范
- 表单验证规则是否符合规范
- 表单设计原则是否遵循（必填项标识、字段分组、验证规则、提示文字等）

#### 2.6 操作按钮规范
- 操作按钮类型是否符合规范
- 操作按钮配置是否符合规范
- 操作按钮规则是否遵循（新增/编辑跳转页面、删除弹窗确认等）

#### 2.7 组件开发规范
- 组件结构是否符合规范（Vue 3 Composition API、<script setup> 语法）
- 组件设计是否遵循单一职责原则
- 组件通信是否使用 props 和 emit
- 组件大小是否控制在合理范围内
- 样式管理是否使用 scoped 样式
- 日期时间选择组件是否符合规范
- 表格组件是否符合规范

#### 2.8 代码风格规范
- 代码格式是否符合规范（缩进、代码长度、引号使用等）
- 命名规范是否符合要求（变量名、函数名、组件名等）
- 注释规范是否符合要求

### 3. 检查方法
- 逐一访问每个页面
- 对照规范文档检查每个检查项
- 记录不符合规范的问题
- 拍摄页面截图作为证据
- 生成详细的检查报告

### 4. 检查工具
- 浏览器开发者工具
- 响应式设计测试工具
- 截图工具
- 检查清单表格

### 5. 检查时间计划
- 准备阶段：1天
- 检查阶段：3天
- 报告阶段：1天

### 6. 输出成果
- 详细的页面合规性检查报告
- 不符合规范的问题清单
- 改进建议

## 优先级
- P0：严重不符合规范，影响用户体验或功能
- P1：中度不符合规范，需要改进
- P2：轻微不符合规范，建议优化

## 注意事项
- 检查时需考虑不同设备尺寸的适配
- 检查时需考虑不同浏览器的兼容性
- 检查时需关注用户体验和操作流程
- 检查结果需客观、详细、可量化
